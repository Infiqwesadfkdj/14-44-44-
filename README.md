## 🚀 Как запустить проект

### 1. Сборка Docker-образа

```bash
# Находясь в папке с Dockerfile и index.html
docker build -t my-nginx-practice:v1 .
2. Запуск контейнера с веб-сервером
bash
# Базовый запуск
docker run -d -p 8080:80 --name my-nginx-container my-nginx-practice:v1
3. Запуск с ограничениями (CPU, RAM, read-only)
bash
docker run -d \
  -p 8080:80 \
  --name my-nginx-secure \
  --memory="50m" \
  --cpus="0.5" \
  --read-only \
  my-nginx-practice:v1
4. Запуск с сохранением логов на хосте
bash
# Для PowerShell (Windows)
docker run -d \
  -p 8080:80 \
  --name my-nginx-with-logs \
  --memory="50m" \
  --cpus="0.5" \
  --read-only \
  -v ${PWD}/nginx_logs:/var/log/nginx \
  my-nginx-practice:v1

# Для Linux/Mac
docker run -d \
  -p 8080:80 \
  --name my-nginx-with-logs \
  --memory="50m" \
  --cpus="0.5" \
  --read-only \
  -v $(pwd)/nginx_logs:/var/log/nginx \
  my-nginx-practice:v1

🛠 Полезные команды Docker
Команда	Описание
docker ps	Показать запущенные контейнеры
docker ps -a	Показать все контейнеры (включая остановленные)
docker images	Показать все образы
docker stop <имя>	Остановить контейнер
docker rm <имя>	Удалить контейнер
docker rmi <образ>	Удалить образ
docker logs <имя>	Посмотреть логи контейнера
docker inspect <имя>	Детальная информация о контейнере
⚠️ Возможные проблемы и их решение
Проблема	Решение
pull access denied for my-nginx-practice	Образ не собран локально. Выполни docker build -t my-nginx-practice:v1 .
Контейнер с --read-only не запускается	Nginx не может писать логи. Добавь volume: -v ${PWD}/nginx_logs:/var/log/nginx
Файлы логов не появляются в nginx_logs/	Проверь права доступа к папке или используй полный путь: -v C:/полный/путь/nginx_logs:/var/log/nginx
Порт 8080 уже занят	Используй другой порт, например -p 8081:80
